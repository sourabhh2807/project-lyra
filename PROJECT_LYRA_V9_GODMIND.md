## 1. THE ACCOUNT STRATEGY

### Why 50 Channels on 1 Account is a Fatal Architecture

The original design of 50 channels under a single Google account is the most
dangerous possible configuration — not a calculated risk, a guaranteed eventual loss.

**The failure mode:** YouTube flags any account showing unusual patterns.
With 50 channels on one account, a single flag → single review → all 50 channels banned
simultaneously. Years of work and compound learning: gone in one decision by one moderator.

### The Correct Architecture: 1 Channel = 1 Google Account

| Rule | Why |
|---|---|
| One YouTube channel per Google Account | Total blast radius isolation |
| Each Google account created on a different IP | Prevents pattern linking at creation |
| Mobile hotspot is the easiest IP switcher | Available to everyone, no VPN cost |
| Never log into two channel accounts simultaneously on the same browser | Cookie-based linking |

### The Scaling Plan

```
Month 1:  3 channels (slots 0, 1, 2)  — Prove the system works
Month 2:  5 channels (add slots 3, 4)  — Add 2 winning-niche variants
Month 3:  8 channels (add slots 5-7)   — Expand on best-performing niches
Month 4: 15 channels                   — Scale to serious volume
Month 6+: 20-25 channels               — Optimal ceiling for quality management
```

**Hard cap recommendation: 20-25 channels maximum.**
Beyond 25, quality signal gets diluted and learning becomes noise.
Depth beats breadth. 5 dominant channels beat 50 average ones every time.

**Trigger to add a new channel:** existing channels reach avg fitness ≥ 0.45 AND
GitHub Actions minutes remain below 1,800/month.

## 2. THE THREE LAWS

These override every evolved strategy, every optimization pressure, every generation.

**Law I — Fitness or Integrity**
Content that underperforms is not just abandoned — its failure is studied, logged, and
converted into permanent knowledge. Content that performs through deception is rejected.
Only honest fitness counts.

**Law II — Zero Local Compute**
Everything runs inside GitHub's free infrastructure. The laptop is needed only for initial
OAuth token generation. After setup, LYRA runs itself indefinitely.

**Law III — Memory is Sacred**
The system grows smarter with every video only if it remembers why things worked and why
they failed. A system with amnesia cannot evolve. Memory is not a feature — it is the
entire foundation.

## 4. THE OBJECTIVE FUNCTION

### Priority Hierarchy (This Order is Law)
1. Viewer satisfaction
2. Watch time quality (AVD %, retention curve shape)
3. Repeat-view potential (return viewer lift)
4. Topic-authority growth (channel depth in niche)
5. Scalable publishing consistency
6. Controlled experimentation yield

### Fitness Score Formula

```
Fitness =
  0.28 × Satisfaction_Score
+ 0.22 × Avg_View_Duration_pct
+ 0.15 × Retention_Stability_Score
+ 0.12 × CTR_Quality_Score
+ 0.10 × Comment_Depth_Score
+ 0.08 × Return_Viewer_Lift
+ 0.05 × Topic_Originality_Score
```

**Satisfaction Score** = like/dislike ratio + saves + shares + positive comment sentiment (LLM-scored)

**Retention Stability Score** = 1 - std_dev(retention_curve). Flat curves beat spikey ones.

**CTR Quality Score** = CTR weighted by AVD. High CTR + low AVD = clickbait penalty.

**Comment Depth Score** = avg comment word count × non-generic ratio.

**Topic Originality Score** = 1 - saturation_score. Oversaturated topics penalized even if they perform.

### Hard Publishing Blocks
Regardless of fitness total, these trigger automatic publish block:
- Factual unreliability flag (LLM fact-checker < 0.70)
- Thumbnail-title mismatch > 0.35
- Repetitive scripting (n-gram overlap with own last 5 videos > 45%)
- Low voice clarity score
- Policy-risk pattern detected

## 5. THE GODMIND INTELLIGENCE LOOP

```
SENSE → FILTER → UNDERSTAND → HYPOTHESIZE → CREATE → QUALITY_GATE
     → PUBLISH → EVALUATE → LEARN → UPGRADE → PRESERVE → [back to SENSE]
```

### Stage 1: SENSE
- Google Trends API (rising queries, breakout topics, seasonal patterns)
- YouTube Data API (trending videos, niche search growth)
- Competitor video analysis (titles, hooks, comment sections)
- Own channel comments (recurring questions, complaints, praise)
- Own retention curves (where attention drops = content gap signal)

### Stage 2: FILTER
Every raw signal passes 5 filters before becoming a topic candidate:
- Relevance filter (fits active niches?)
- Novelty filter (covered this angle in last 60 days?)
- Saturation filter (overserved with no differentiation gap?)
- Credibility filter (can we cover this accurately?)
- Brand fit filter (aligns with channel identity?)

### Stage 3: UNDERSTAND
LLM constructs a structured understanding object for each surviving topic:
- Why do people care about this now?
- What emotional state is the viewer in when they search this?
- What do they already know? What do they assume incorrectly?
- What confusion remains unanswered in existing content?
- What is the viewer's identity context?
- What specific promise must the video make and fulfill?

### Stage 4: HYPOTHESIZE
Explicit testable hypotheses generated before creation:
- "Contrarian hook will outperform curiosity gap for this topic"
- "Long-form will beat Shorts here because audience is solution-aware"
- "Comparison narrative will outperform list for this niche segment"

Hypotheses stored. Outcomes confirm or deny. Confirmed → rules. Denied → anti-patterns.

### Stage 5: CREATE
Full content package generated using champion genome + topic understanding:
- Long-form script (10-15 min) with scene-by-scene structure
- Shorts script (45-58 sec) with rapid payoff structure
- 3 title candidates (scored against patterns)
- 3 thumbnail concepts (scored by CTR predictor)
- Edit pacing plan (scene intensity map)
- Full metadata (description, tags, optimal upload time)

### Stage 6: QUALITY GATE
All 7 gates must pass. One failure = hold for revision or discard.
Never publish a failure. See Section 9 for full gate specifications.

### Stage 7: PUBLISH
Upload long video + Shorts via YouTube Data API v3.
Set thumbnail. Post seed comment. Log everything.

### Stage 8: EVALUATE
48h post-upload analytics pull.
Compute composite fitness. Generate derived metrics. Compare against hypothesis.

### Stage 9: LEARN
Convert outcome into reusable knowledge objects:
- Confirmed hypothesis → promoted rule → long-term memory
- Denied hypothesis → anti-pattern → failure memory
- Unexpected result → anomaly flag → investigation queue

### Stage 10: UPGRADE
Controlled gene mutation. Only with evidence threshold met.
Only one layer at a time. Always with rollback. See Section 11.

### Stage 11: PRESERVE
Write all new knowledge to versioned storage.
Commit to repo. Increment generation counter. Nothing lost.

## 6. THE HUMAN BEHAVIOR MODEL

Human content consumption is a six-dimensional cognitive event — not a stimulus-response reflex.

### The Six Dimensions

**Dimension 1 — Attention (Why They Stop Scrolling)**
Triggered by: novelty, pattern interrupt, identity relevance, unresolved tension.
Design principle: First 2 seconds must contain ONE of these. Not all. One, precisely executed.

**Dimension 2 — Curiosity (Why They Keep Watching)**
Curiosity = information gap + perceived value of resolution.
Design principle: Open a loop early. The resolution must feel worth 10+ minutes of their life.

**Dimension 3 — Comprehension (Whether They Understand)**
Confusion = exit. Comprehension requires: correct knowledge baseline assumption,
clear logical flow, no jargon without definition, visual reinforcement of key concepts.
Design principle: Write for the smartest version of your target viewer, not the average.

**Dimension 4 — Tension (Whether Anticipation Is Maintained)**
Tension = knowing something important is coming but not yet knowing what.
Design principle: Maintain at least one unresolved thread at every 90-second mark.

**Dimension 5 — Reward (Whether the Promise Is Fulfilled)**
The thumbnail and title make a promise. The video must pay it off completely and slightly exceed it.
Design principle: The 10-second window after the main reveal drives likes, saves, and return views.

**Dimension 6 — Identity (Whether They Feel It Is "For People Like Me")**
The most underestimated dimension. Viewers share content that reflects their self-image.
They subscribe when the channel understands their specific worldview.
Design principle: Every niche has an identity archetype. Content must speak that archetype's language.

### Audience State Model

| State | Description | Content Strategy |
|---|---|---|
| Unaware | Doesn't know the problem/topic exists | Lead with consequence or surprise |
| Problem-aware | Knows problem, doesn't know solutions | Lead with empathy, then reveal |
| Solution-aware | Knows solutions exist, evaluating options | Lead with differentiation |
| Comparison mode | Actively comparing options | Lead with honest analysis |
| Emotionally charged | Strong emotional state about the topic | Lead with validation |
| Entertainment-seeking | No specific need, wants engagement | Lead with personality and energy |

### Behavioral Triggers Library
Stored in `intelligence/behavior_models/triggers.json`. Win-rate tracked per niche + audience state.

```
curiosity_gap       | fear_of_missing_out   | status_gain
surprise_reversal   | specificity           | practical_utility
emotional_recognition | taboo_revelation    | conflict_framing
mastery_progression | identity_callout      | contrarian_claim
transformation_story | insider_knowledge    | myth_busting
```

Rule: One strong trigger per video, supported by narrative. Five triggers fighting each other = zero triggers.

## 7. THE 24/7 RESEARCH ARCHITECTURE

Research never stops. Creation is selective. Research is relentless.

### Research Agent Responsibilities

**A. Trend Sensing**
- Google Trends API: rising queries, breakout topics, seasonal patterns, geographic variance
- YouTube Data API: niche-specific trending, velocity of view growth
- Rising compound queries (e.g., "dark history + psychology" emerging together)
- Seasonal recurrence: same calendar window performance from prior years

**B. Competitor Intelligence**
For each niche, monitor top 20 competitor channels:
- Topic clusters (subjects they keep returning to)
- Title formula patterns (extracted and stored as alleles)
- Hook format patterns (first 10 seconds analyzed by LLM)
- Comment section mining (complaints = gaps, questions = demand, praise = replicate)
- Underperforming video comments (what disappointed viewers expected)

Critical Rule: Extract principles and patterns. Never clone content. Learn WHY it works.

**C. Audience Intelligence**
Continuously updated audience maps per niche:
- Fears (what they are afraid of related to this niche)
- Goals (what they are trying to achieve)
- Frustrations (what existing content fails to give them)
- Language patterns (vocabulary, phrases, cultural references they use)
- Identity signals (who they see themselves as)

**D. Knowledge Gap Detection**
- Unanswered "how/why/what about X" patterns appearing ≥5 times in comments
- Contradiction areas (two popular videos claiming opposite things = opportunity)
- Misinformation correction opportunities (wrong info widely believed = authority content)

**E. Saturation Detection**
Before any topic enters creation queue:
- Number of existing videos on exact topic (last 90 days)
- Quality of top 5 existing videos (high quality = high saturation)
- View velocity of existing videos (slowing = late, accelerating = still open)
- Whether LYRA has a meaningfully differentiated angle

Saturation > 0.75 without differentiation = topic rejected.

## 8. THE GENOME — FULL GENE MAP

Every gene represents a meaningful content decision backed by evidence.
Every gene tracks its own win rate, niche-specific performance, and interaction effects.

### Gene Categories

**Topic Genes** — Strategic content decisions
- `niche` — assigned niche
- `urgency_type` — trending | timeless | seasonal | reactive
- `novelty_type` — first-coverage | new-angle | myth-bust | deep-dive
- `authority_fit` — 0.0–1.0 alignment with channel history
- `audience_state_target` — which viewer state this targets

**Hook Genes** — First 5 seconds
- `hook_type` — shock_statement | curiosity_gap | specificity | identity_callout | contrarian_claim | transformation_promise | question_hook | statistic_shock
- `hook_duration_sec` — 2.5–6.0
- `open_loop_count` — 1–3
- `hook_visual_intensity` — low | medium | high

**Narrative Genes** — Story structure
- `narrative_structure` — documentary_reveal | myth_busting | transformation_story | case_study | comparison | listicle_elevated | deep_dive | controversy_analysis
- `payoff_placement_pct` — 0.55–0.80
- `escalation_curve` — flat | linear | exponential | spike_late

**Delivery Genes** — How it sounds
- `narration_style` — authoritative_dramatic | conversational_engaging | teacher_clear | storyteller | provocateur | investigative
- `narration_pace_wpm` — 130–190
- `voice_energy_level` — calm | medium | high
- `humor_intensity` — none | subtle | moderate
- `directness_level` — indirect | direct | blunt

**Visual Genes** — How it looks
- `pacing_profile` — fast_hook_slow_reveal | constant_medium | cinematic | rapid_fire | build_and_release
- `motion_density` — cuts per minute (adaptive)
- `overlay_density` — text overlays per minute
- `illustration_ratio` — illustration vs stock footage pct
- `color_grade` — niche-specific palette

**Packaging Genes** — How it is presented
- `title_archetype` — 12 defined patterns in `title_patterns.json`
- `thumbnail_style` — 8 defined layout templates
- `promise_style` — specific | broad | question | statement
- `payoff_framing` — transformation | revelation | correction | validation

**Trust Genes** — Whether viewers believe it
- `source_citation_density` — none | light | moderate | heavy
- `confidence_wording_style` — hedged | direct | assertive | authoritative
- `explanation_depth` — surface | medium | deep

### Gene Interaction Effects (Critical)
The system tracks not just individual gene performance but combinations:

```json
{
  "gene_pair": "hook_type=curiosity_gap + narrative_structure=documentary_reveal",
  "interaction_effect": +0.12,
  "sample_size": 34,
  "niche": "dark_history"

```

This combinatorial intelligence cannot be replicated by human creators tracking manually.

## 9. THE CONTENT QUALITY ENGINE

**No content publishes without passing all 7 gates.**
A failed gate holds the content for revision or marks it for discard.

### Gate 1 — Research Quality
- Topic still timely (urgency_decay not exceeded)
- Angle differentiated from own last 30 videos
- Audience need explicit and documented
- Research confidence ≥ 0.35

### Gate 2 — Script Quality
- Clear promise within first 30 words
- Logical flow throughout
- Fluff ratio < 15%
- No redundant phrasing (n-gram check)
- Strong payoff — conclusion exceeds hook promise slightly
- Factual accuracy ≥ 0.70

### Gate 3 — Narrative Quality
- Hook present and effective
- Setup establishes stakes
- Escalation maintains tension
- Reveal/payoff satisfying, not anticlimactic
- Clean close or opens future loop

### Gate 4 — Audio Quality
- Voice clarity ≥ 0.65 (SNR check via librosa)
- Pacing variety detected (not monotone WPM)
- No robotic monotony (pitch variance check)
- Music at -18 LUFS or below when narration present

### Gate 5 — Visual Quality
- Every scene change increases clarity or intensity
- No static image held > 8 seconds without motion or text change
- Scene density adapts to content type

### Gate 6 — Authenticity Quality
- Discernible point of view (not just Wikipedia summary)
- No fake imperfection tricks
- Consistent voice identity throughout
- Sounds like a position, not a template

### Gate 7 — Platform Safety & Trust
- No deceptive framing
- Thumbnail-title honesty ≥ 0.65
- No prohibited content patterns
- No misleading statistics

### Adaptive Pacing Rule

```
Pacing = f(content_intensity, explanation_density, emotional_moment, retention_risk)

Fast pacing   → hooks, transitions, high-energy moments
Medium pacing → examples, comparisons, context-building
Slow pacing   → key reveals, emotionally significant lines, important data
```

## 10. THE FREQUENCY SYSTEM

### Four Content Classes

| Class | Description | Action |
|---|---|---|
| Core Content | High fitness prediction + all gates pass | Publish at optimal time |
| Experimental | Hypothesis test, specific gene variation | Publish to 1 channel, monitor |
| Draft Bank | Generated but held — timing or minor revision needed | Queue for evergreen bank |
| Evergreen Reserve | Timeless, high quality, no urgency | Store for remix cycle |

### Dynamic Cadence Model

```
Publish if:
  fitness_prediction ≥ quality_threshold
  AND all 7 gates pass
  AND channel NOT in audience_fatigue_zone
  AND topic within freshness_window
  AND rendering_capacity available
```

### Target Cadence (Not a Hard Quota)
- 1 long video per day per channel (10-15 min)
- 1-2 Shorts per day per channel (45-58 sec)
- Quality gate may reduce this — that's correct behavior, not failure

### Audience Fatigue Detection
Frequency Controller monitors:
- Fitness trend over last 5 videos (>20% decline = fatigue signal)
- Comment tone shift (from engaged to generic)
- Return viewer rate change

If fatigue detected: reduce cadence 40%, increase quality threshold 20%.

## 11. THE SELF-EVOLUTION ENGINE

### Two-Layer Evolution

**Layer A — Content Evolution (Every 48h)**
- Hook types, thumbnail styles, narrative structures, pacing profiles, title formulas
- Requires: ≥50 video sample, expected upside exceeds risk threshold, holdout test passing

**Layer B — System Evolution (Monthly, strict approval)**
- Fitness function weights, topic selection heuristics, mutation rates, promotion thresholds
- Requires: ≥200 video sample, 30-day window, rollback version preserved, explicit version increment

```json
    "sample_size": 85,

  "holdout_passed": true

```

### Champion-Challenger Architecture
- Champion (current best gene set) = 80% of upload volume
- Challenger (new hypothesis) = 20% for minimum 2-week test window
- Promote Challenger only if it beats Champion across full window, never on single outlier

### ε-Greedy Bandit (Niche Allocation)
- ε starts at 0.30 (30% of compute to unexplored niches)
- Decays 0.005 per evolution cycle toward minimum of 0.05
- At ε=0.05: 5% exploration, 95% exploitation of best-performing niches

## 13. THE ANALYTICS INTELLIGENCE LAYER

### Required Raw Metrics
- Impressions, CTR, Average view duration (seconds + percentage)
- Full retention curve (not just average)
- Likes, saves, comment count + sample for depth scoring
- Subscriber change post-video
- Return viewer rate change
- Traffic source breakdown (browse = algorithm push, search = evergreen)

### Derived Metrics (Computed by `analytics_agent.py`)

**Hook Strength Score** = AVD at 0:30 relative to channel baseline

**Mid-Video Collapse Score** = Steepest retention drop between 30% and 80% of video

**Payoff Fulfillment Score** = Retention at 90% relative to 70% of video
Viewers staying through the end = promise delivered.

**Pacing Overload Score** = cuts/min ÷ explanation_density
High cuts during complex explanation = viewer confusion signal.

**Novelty Fatigue Score** = Fitness decay trend across last 5 videos with same narrative_structure gene.
Indicates the structure wearing out in this niche.

**Title-Thumbnail Honesty Score** = LLM comparison of what thumbnail+title promises vs video delivers.
Below 0.65 = publishing block.

## 18. GROWTH, ENGAGEMENT & TRUST

### Thumbnail Strategy
1. Generate base image from Pollinations (subject + emotional context)
2. Darken image 45% for text legibility
3. Apply niche color grade (stored in `niche_config.json`)
4. Bold text overlay via Pillow — max 4 words, contrasting color
5. Left-side accent bar in niche color
6. Honesty check: LLM compares thumbnail promise vs script payoff (must be ≥ 0.65)

### Title Engine
- Generate 5 candidates using active `title_archetype` gene
- Score each: specificity, curiosity gap strength, identity relevance, promise clarity

- Select top scorer that passes honesty gate

- Draft stored in `creation/packaging/` as pending approval
- Operator approves via Telegram bot (one tap) → bot sends via YouTube API

- System seeds one pinned comment per video as channel owner (standard creator practice)

`upload_agent.py` tracks optimal upload time per niche per channel.
Starts with evidence-based defaults. Updates every 30 days from analytics.

Default starting times (UTC):

## 19. SECURITY, RESILIENCE & TOS SAFETY

### API Key Security
- All secrets in GitHub Actions Encrypted Secrets (never in code)
- Separate OAuth token per channel (`YT_OAUTH_TOKEN_0` through `YT_OAUTH_TOKEN_N`)
- Gemini and Groq keys rotated every 60 days
- No secrets ever logged (GitHub masks them automatically in run logs)

### Channel Isolation
- One OAuth token per channel — if one is flagged, others unaffected
- `channel_map.json` status field: `active | paused | struck | terminated`
- Struck channels auto-paused in next cycle
- Struck channel alleles quarantined to `reflection/postmortems/` for analysis

### Resilience Chain
```
Gemini fails → Groq fallback
Groq fails   → Skip LLM-dependent step, use template
Pollinations fails → HuggingFace SDXL fallback
HF fails     → Pillow placeholder frame
Edge-TTS fails → asyncio edge-tts API fallback
Assembly fails → Mark genome failed_assembly, retry next cycle
Upload fails → Mark genome upload_pending, retry next cycle
Any crash    → try/except logs to error_log.json, pipeline continues
```

### Absolute ToS Commitments (Non-Negotiable)
1. No fake views, view farms, or purchased engagement
2. No bot interactions on competitor channels
3. No automated comments from fake or external accounts
4. No resolution/framerate tricks for hash evasion
5. No content designed to deceive viewers
6. Thumbnail-title must honestly represent content (hard gate enforces this)
7. No prohibited content in any category
8. No scraping that violates robots.txt of any platform

## 26. KEY DESIGN DECISIONS

**Why start with 3 channels instead of 50?**
50 channels on 1 Google account is catastrophic risk. 3 channels on 3 separate accounts provides
blast radius isolation, clean learning signal per niche, and manageable OAuth overhead.
Scale after the system proves itself. Depth before breadth.

**Why SQLite in a Git repo instead of a real database?**
Zero cost, zero ops, version-controlled schema, atomic commits.
For ≤25 channels and ≤50,000 genomes across 24 months, SQLite is faster and simpler.

**Why Edge-TTS over gTTS?**
gTTS sounds robotic after 30 seconds — viewers associate robotic voice with low-effort content
and disengage faster. Edge-TTS produces Microsoft Neural voices with near-human quality.
Voice quality is a trust gene. It determines how long viewers stay.

**Why long-form videos (10-15 min) AND Shorts?**
Long-form builds authority, drives AVD, and earns algorithm trust over time.
Shorts drives discovery and subscriber velocity. They are not competitors — they are a funnel.
The same genome generates both: same topic, same research, same brand voice.

**Why not use n8n, Make, or any scheduling service?**
GitHub Actions IS the scheduling service. Zero external dependencies means zero additional
failure points. The entire brain lives in one repository.

**Why the 6-dimension human behavior model instead of retention hacks?**
Hacks work once. Understanding works permanently. The system that understands WHY humans
engage can adapt when any single tactic becomes overused. The six dimensions are stable
psychological truths, not platform-specific exploits.

**Why preserve failures as aggressively as successes?**
A genome that only learns what worked will eventually over-exploit winning patterns until they
saturate and fail catastrophically. The system needs to know what fails, why it fails, and
what gene combinations produce failure even with otherwise good DNA. Failure memory is the
system's immune system.
